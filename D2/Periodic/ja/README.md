# LUX.Delaunay.D2.Periodic

[English](../README.md)

**周期境界**を持つ２Ｄドロネー図 ―― 平坦トーラス `T = [0,L)²` 上のドロネー三角形分割。ゴースト点の複製も被覆空間（3×3コピー）も一切使わず、**常に最小表現（頂点 n 個・面 2n 枚）**を保つ。n = 1 でも「1頂点・2面」のΔ複体（同じ頂点の実体を異なる格子オフセットで面が3回参照する）として成立する。2009年の `VoronoiTools2P`（Jump / Shift を持ち回る周期 Bowyer-Watson）の方式を、厳密述語と一般ケースの完全な取り扱いによって洗練し直したものである。

## クラス

| クラス | 基底 | 役割 |
|---|---|---|
| `TPeriPoin2D` | `TTriPoin2D<TPeriFace2D>` | 頂点（サイトと1対1。`Site` 番号を持つ） |
| `TPeriPoinSet2D` | `TTriPoinSet2D<TPeriPoin2D>` | 点集合 |
| `TPeriFace2D` | `TTriFace2D<TPeriPoin2D,TPeriFace2D>` | 三角形（角ごとの格子オフセット・リフト幾何） |
| `TPeriFaceSet2D` | `TTriFaceSet2D<TPeriFace2D,TPeriPoinSet2D>` | 面集合 |
| `TPeriDelaunay2D` | `TPeriFaceSet2D` | モデル：サイトの追加・削除 |

TriFlip の接続構造（Poin / Face / Corn）は頂点の同一性ではなく角番号の組合せで巡回するため、Δ複体をそのまま載せられる。TriFlip 本体は無変更で、周期固有の情報だけを派生クラスが加える。

## アルゴリズムの要点

- **格子オフセットと正準化** ―― 頂点座標は常に正準（∈ [0,L)²）。面は角ごとにオフセット o ∈ {0,1,2}（2bit×2軸）を持ち、角の幾何座標は `Pos + o·L`。面ごとに普遍被覆 R² への独自のリフトを持つ（隣接面との変換は `NeigShift`）。オフセットは面の生成時に必ず正規化するため、**追加・削除を繰り返しても実体が基本領域から乖離することはなく、「散らばった実体を巻き戻す」処理は原理的に不要**。
- **量子化と厳密整数述語** ―― L を2の冪グリッド q = 2^(E−17)（L/q ∈ [2^16, 2^17]）へスナップし、座標を q の整数倍に量子化する（誤差 ~L·2⁻¹⁶、実用上不可視）。全幾何が整数格子に載るため、向き判定は64bit整数で、空円判定は**128bit整数の累算で厳密に**評価される。丸め誤差による構造破壊が原理的に存在しない。
- **記号摂動（サイト順位 SoS）** ―― 共円の退化は、サイト順位によるリフトの無限小摂動（無限小重み付きドロネーと等価）で決定的に裁く。摂動はサイト単位（全周期像に共通）なので**格子同変**であり、「同一頂点の平行移動対 (w, w+L·e) による構造的な共円」も解消される。それでも決まらない超退化（例：厳密に対称な格子点配置の疎な段階）だけが拒否される（`AddPoin` = nil / 削除 = 再構築へ退避。分割は常に正しいまま保たれる）。
- **追加（普遍被覆上の Bowyer-Watson）** ―― 追加点のリフト p̂ を固定し、キャビティを（面の実体 × 格子平行移動）の対として幅優先で集める。同じ面が異なる平行移動で2回入るケース（外接円が p の複数の周期像を含む）も正しく扱う。
  - 通常（p が自分の周期像とドロネー隣接しない場合。厳密判定つき）：境界辺への**錐張り**。
  - 疎な配置：錐は正しくない（自己辺 p–p を持つ面が必要）ので、**p̂ のスターを候補点集合からギフトラッピングで直接構築**し、トーラスへの射影で重複する面を回転・平行移動正規化キーで同一視し、縫合は幾何計算で一意に解決する。計画が完全に検証されてからメッシュに触れる。
- **削除（局所・星の除去と耳刈り）** ―― 頂点のひとつのリフトの周りの星を角の巡回で集め、穴の境界多角形を**ドロネー耳**（リンク頂点とその平行移動像を外接円に含まない耳）で埋める。穴がトーラスを巻いて自分の平行移動像と接する場合は、隣の穴の平行移動 μ を幾何的に特定して埋め草どうしを縫う。計画検証後にのみメッシュへ反映し、退化配置で埋め戻せなければ**何も変えずに False** を返す（平面版と同じ流儀）。サイト数 ≤ 3 だけは O(1) の再構築で扱う（平面版の少数点の特別扱いに相当）。実測では一様乱数200点の全削除で **197/200 が局所削除**（残り3回は末尾の n ≤ 3）。
- 位置検索はジャンプ＆ウォーク（累積平行移動・厳密述語・全面走査の保険つき）。

## API の概要

```pascal
var D :TPeriDelaunay2D;

D := TPeriDelaunay2D.Create;
D.Size := 300;                                  // 基本領域 [0,300)²

V := D.AddPoin( TSingle2D.Create( X, Y ) );     // 領域内へ巻き込まれる。重複・超退化は nil
D.FindNearPoin( P, V );                          // 最近傍サイト（トーラス距離）
D.DeletePoin( V );                               // 局所削除（退化配置では何も変えずに False。n ≤ 3 は O(1) 再構築）

for F in D.Faces do                              // 面 = トーラスの面そのもの（常に 2n 枚）
   ... F.CornPos( K ) ... F.CircumPos ... F.NeigShift( K ) ...

D.LocalDelN;  D.RebuildDelN;  D.StarInsN;        // 統計（局所削除／再構築退避／スター構築の回数）
D.OnChange.Add( Handler );                       // 変更通知
```

## 注意・制限

- 領域は**正方形**トーラス（`Size × Size`）。
- 厳密に共円な退化配置（グリッド上の対称配置など）は、疎な段階（スター構築が要る局面）に限り追加が拒否されることがある。密になれば（錐張りで済む通常状態）拒否は起きない。
- TriFlip コンテナの `SaveToFile` / `LoadFromFile` は封じてある（形式が格子オフセットを保存できないため）。
- `AddPoin` / `FindNearPoin` が返す頂点参照は、その頂点の削除で無効になる ―― すぐに使うこと。

## ビューア

`LUX.Delaunay.D2.Periodic.Viewer`（`TPeriDelaunayViewer`）は LUX.CG2D のシーングラフでモデルを描く。カメラ視野は 2L×2L、基本領域を原点中心に置き、トーラスの面を固定の 3×3 タイルで貼り並べる。レイヤは上から：頂点（赤）→ ボロノイ辺（黒）→ 周期境界の直線（ピンク・3×3 の縦4本横4本）→ 空円（緑）→ ドロネー三角形（水色）。頂点・三角形は**中央の実体を濃色、周囲8コピーを50%**で描く（実体用・コピー用の別レイヤに分け、淡色は本体レイヤの色から毎フレーム導出する。中間ノードにスタイルを継承させる方式はこのシーングラフでは子へ伝播しないため使わない）。空円は基本領域の実体のぶんだけを描く（周囲8タイルのコピー円は描かない）。実体面の配置はアンカー（角1）を基本領域へ巻き戻すだけの単純規則とする。面ごとの局所規準（外心・重心など）で中央代表を辺連結な塊に見せることは一般に不可能で、連結な基本領域の選択は双対グラフ上の全域木選択と同型の最適化になるため、行わない。
